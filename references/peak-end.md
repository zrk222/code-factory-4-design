# Peak-End Rule — playbook (the "Delight Hunt")
Users remember peaks + the ending, not the average.
- Peak audit: find 3–5 high-intent moments (CTA click, form completion, purchase).
- Peak generation: satisfying micro-interactions — hover elevation/color, smooth `transition:all .3s ease`.
- Inline form validation: `:valid`/`:invalid`, gentle green checkmark = "gentle confirmation".
- Polish overlooked states: real loading + empty states, not generic spinners.
- The end: rewarding, transparent confirmation screen with a clear sense of achievement.
- Never end on a negative peak (surprise fee, dead-end).
